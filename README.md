# Goat Speckle — сайт маникюрной студии

Простой статический сайт на чистом HTML/CSS/JS (без фреймворков и сборки).
Состоит из 5 страниц: Главная, О нас, Услуги, Новости, Контакты.

## Структура проекта

```
goat-speckle/
├── index.html        — главная страница
├── about.html         — о нас
├── services.html      — услуги и цены
├── news.html          — новости
├── contacts.html       — контакты + форма записи
├── css/
│   └── style.css
├── js/
│   └── script.js
├── images/
│   ├── logo.jpeg        — исходный логотип
│   ├── logo-nav.jpg      — логотип для шапки/подвала
│   ├── logo-hero.jpg     — логотип для главного экрана
│   └── favicon-256.png   — иконка вкладки браузера
└── vercel.json
```

## Как открыть сайт локально (в VS Code)

1. Установите расширение **Live Server** в VS Code (Ritwick Dey).
2. Откройте папку `goat-speckle` в VS Code (`Файл → Открыть папку`).
3. Кликните правой кнопкой по `index.html` → **Open with Live Server**.
4. Сайт откроется в браузере на `http://127.0.0.1:5500` — можно смотреть все страницы и проверять правки в реальном времени.

Без Live Server тоже можно — просто дважды кликнуть по `index.html`, он откроется в браузере (но некоторые вещи, например форма с относительными путями, лучше проверять именно через Live Server).

## Как разместить сайт в облаке (бесплатно)

### Вариант 1 — Vercel (рекомендуется, проще всего)

1. Зарегистрируйтесь / войдите на [vercel.com](https://vercel.com) через GitHub.
2. Создайте новый репозиторий на GitHub и залейте туда папку `goat-speckle`:
   ```bash
   cd goat-speckle
   git init
   git add .
   git commit -m "Первая версия сайта Goat Speckle"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/goat-speckle.git
   git push -u origin main
   ```
3. В Vercel нажмите **Add New → Project**, выберите этот репозиторий.
4. Framework Preset — оставьте **Other** (это статический сайт, сборка не нужна).
5. Нажмите **Deploy**. Через минуту сайт будет доступен по адресу вида
   `https://goat-speckle.vercel.app`.
6. При каждом новом `git push` в ветку `main` сайт будет обновляться автоматически.
7. Свой домен можно подключить в настройках проекта → **Domains**.

### Вариант 2 — GitHub Pages (тоже бесплатно)

1. Залейте проект в репозиторий на GitHub (см. команды выше).
2. В репозитории откройте **Settings → Pages**.
3. В разделе **Source** выберите ветку `main` и папку `/root`.
4. Сохраните — через 1-2 минуты сайт будет доступен по адресу
   `https://ВАШ_ЛОГИН.github.io/goat-speckle/`.

### Вариант 3 — Netlify

1. Зарегистрируйтесь на [netlify.com](https://netlify.com).
2. **Add new site → Import an existing project**, подключите GitHub-репозиторий.
3. Build command оставьте пустым, Publish directory — `.` (корень).
4. Нажмите **Deploy site**.

## Форма обратной связи и админка

Форма на странице «Контакты» сохраняет заявки в базу данных Supabase.
Если Supabase не настроен (ключи не заполнены) — форма автоматически
переключается на резервный вариант через `mailto:`.

### Настройка Supabase (один раз)

1. Зарегистрируйтесь на [supabase.com](https://supabase.com) и создайте новый проект.
2. Откройте **SQL Editor** в проекте, вставьте содержимое файла
   `supabase/schema.sql` из этого репозитория и нажмите **Run** —
   это создаст таблицу `bookings` с нужными правами доступа.
2.1. Затем вставьте и выполните `supabase/schema_02_content.sql` —
   это создаст таблицы `services` (услуги/цены) и `news` (новости)
   и сразу наполнит их текущим контентом с сайта.
3. В **Authentication → Users** создайте пользователя (email + пароль) —
   это будут ваши данные для входа в админку.
4. В **Project Settings → API** скопируйте **Project URL** и **anon public key**.
5. Откройте файл `js/supabase-config.js` и вставьте эти два значения
   вместо `ВСТАВЬТЕ_СЮДА_PROJECT_URL` и `ВСТАВЬТЕ_СЮДА_ANON_KEY`.
6. Сохраните, закоммитьте и запушьте изменения — форма и админка заработают.

### Как попасть в админку

Откройте `ваш-сайт.vercel.app/admin/login.html` (или ссылку «Админка»
в подвале любой страницы сайта) и войдите с email/паролем, которые вы
создали в Supabase на шаге 3 выше.

В админке три раздела (вкладки наверху):

**Заявки** (`admin/dashboard.html`)
- Список всех заявок с сайта (имя, телефон, комментарий, дата)
- Смена статуса заявки: новая / в работе / выполнена / отменена
- Живое обновление списка при поступлении новой заявки
- Счётчики по количеству заявок в каждом статусе

**Услуги и цены** (`admin/services.html`)
- Добавление, редактирование и удаление услуг без правки кода
- Группировка по категориям (например «Маникюр», «Педикюр» — если добавите новую категорию, на сайте появится отдельная таблица)
- Порядок отображения внутри категории (поле «Порядок отображения»)
- Возможность скрыть услугу с сайта, не удаляя её (галочка «Показывать на сайте»)
- Изменения сразу видны на странице «Услуги» — обновление страницы не требуется публиковать заново

**Новости** (`admin/news.html`)
- Добавление, редактирование и удаление новостей
- Дата публикации, заголовок, текст, необязательная ссылка «Подробнее»
- Возможность скрыть новость с сайта, не удаляя её

Если база данных временно недоступна — страницы «Услуги» и «Новости»
на сайте покажут актуальный контент на момент последней публикации
(зашитый прямо в HTML как резервный вариант), сайт не сломается.
